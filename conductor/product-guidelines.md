# Product Guidelines

## Development Philosophy
- **Idiomatic/Standard**: We follow established patterns and conventions. Consistency and reliability are prioritized to ensure the project remains accessible to the open-source community.
- **Modular/Extensible**: The architecture must remain highly decoupled. New features should be added as independent modules, allowing for focused, iterative development by a single contributor.

## Documentation & Prose Style
- **Technical/Concise**: Communications should be direct and accurate. We focus on clarity to facilitate efficient maintenance.
- **Educational/Detailed**: Complex decisions must be thoroughly documented. This ensures that the "why" behind the code is preserved for future reference and external contributors.

## User & Developer Experience (UX/DX)
- **Performance-First**: High speed and minimal latency are critical. Tools should be lightweight and responsive.
- **Open-Source/Individual Focus**: The branding and deliverables reflect a lean, single-user open-source project. We prioritize functional robustness and rapid iteration over enterprise-scale ceremonial overhead.

## Quality Standards
- **Strict Isolation**: All operations must respect the project root isolation to prevent environment contamination.
- **Pragmatic Verification**: No code change is complete without functional verification. We focus on high-impact tests that provide the most value for the development effort.
