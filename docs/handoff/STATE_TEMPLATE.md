   ```bash
   source docs/handoff/context-guard.sh
   ```
3. **Check which phase to execute** based on the Status field
4. **Log decisions** as you make them — add rows to the Decision Log table
5. **Log changes** as you make them — add rows to the Changes table
6. **Run gate checks** using the context-agnostic framework:
   ```bash
   source docs/handoff/context-agnostic-gates.sh
   phaseN_gate  # Replace N with your phase number
   ```
7. **Update this file** with completion status, gate check output, and any issues

### Decision Log Format

| Column | What to Record |
|--------|---------------|
| **Decision** | What you chose to do |
| **Rationale** | Why you chose it (evidence-based) |
| **Alternatives Considered** | What else you evaluated and why you rejected it |
| **Outcome** | What actually happened |

### Changes Format

| Column | What to Record |
|--------|---------------|
| **File** | Path relative to project root |
| **Action** | Created, Updated, Deleted, Verified |
| **Before** | State before the change |
| **After** | State after the change |
| **Commit SHA** | Git commit that contains the change |
