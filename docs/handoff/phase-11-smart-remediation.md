## Step 11.3 — Add Branch Name Input Validation (R3)

**Issue:** Branch names used directly in git commands without pattern validation. Add validation function to `modules/safety.sh`.

**Action:** Add this function to `modules/safety.sh`:

```bash
# Validate branch name format
validate_branch_name_format() {
    local branch_name="$1"

    # Allow alphanumeric, hyphens, underscores, dots, and forward slashes
    # But not starting with slash, dot, or containing dangerous sequences
    if [[ "$branch_name" =~ ^[a-zA-Z0-9][a-zA-Z0-9._/-]*$ ]] && \
       [[ ! "$branch_name" =~ [\$\`\(] ]] && \
       [[ ! "$branch_name" =~ ^/ ]] && \
       [[ ! "$branch_name" =~ \.\. ]]; then
        return 0
    else
        echo "ERROR: Invalid branch name format: $branch_name"
        echo "Branch names must start with alphanumeric character and contain only [a-zA-Z0-9._/-]"
        return 1
    fi
}

export -f validate_branch_name_format
```

Then call this function in `modules/branch.sh` before any git operations that use user-supplied branch names.

**Verify:**
```bash

validate_branch_name_format "orchestration-tools-feature" && echo "PASS: valid" || echo "FAIL"
validate_branch_name_format 'branch$(rm -rf /)' && echo "FAIL: should reject" || echo "PASS: rejected"
validate_branch_name_format "../escape" && echo "FAIL: should reject" || echo "PASS: rejected"
```

PASS: rejected
