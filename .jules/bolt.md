## 2024-03-24 - [Implement Search Debounce in Dashboard]
**Learning:** Found a missing performance pattern where search inputs triggered API calls on every keystroke, which can overload the backend and cause frontend sluggishness. Also discovered that `client/src/components/advanced-filter-panel.tsx` was using an alias path (`@components`) that wasn't resolving correctly with standard TS compilation (`tsc`), while other files like `dashboard.tsx` used the standard alias (`@/components`).
**Action:** When implementing similar UI components with frequent inputs (like search or filters), ensure that debouncing or throttling is applied by default. Additionally, consistently enforce the use of standard `@/` alias paths across the React components to maintain build consistency and avoid TypeScript resolution errors.
