## 2025-06-16 - [Debounce Search Input in React Query]
**Learning:** In a codebase leveraging React Query (`useQuery`) with dependencies mapped directly to component state (like a raw `searchQuery` from an `<Input>`), each keystroke triggers a refetch because the `queryKey` updates instantly. This causes rapid, unnecessary re-renders and excessive network requests, which is a common frontend performance bottleneck when search inputs are unoptimized.
**Action:** Implemented a standard debounce hook (`useEffect` with a 300ms `setTimeout`) to create a `debouncedSearchQuery` state. We then bind the `useQuery` query key to the debounced state instead of the raw state. This simple frontend adjustment ensures only purposeful, delayed input triggers expensive data fetching or refetching.
